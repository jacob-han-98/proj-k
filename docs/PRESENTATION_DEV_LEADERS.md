## 1. 왜 만드는가

### 현재 기획 지식 관리의 문제

- 기획서가 **Excel 104개, Confluence 490페이지, PPT 11개**에 분산
- 기획자 한 명이 전체 시스템을 파악하는 것이 사실상 불가능
- 시스템 간 충돌이나 누락을 사전에 발견하기 어려움
- 핵심 기획 지식이 특정 인원의 머릿속에만 존재 (퇴사 시 유실)
- 신규 입사자 온보딩에 수 주 소요

### 해결 방향

**AI가 모든 기획 문서를 구조화하여 보유하고, Project K의 기획을 가장 잘 아는 전문가처럼 기획자의 작업을 실시간으로 보조한다.**

## 3. 지금까지 만든 것 — 1단계: 지식화

### 기획서 → AI가 읽을 수 있는 구조화된 지식으로 변환

**변환 대상 및 결과**:

| 유형 | 원본 | 변환 결과 | 비고 |
|------|------|-----------|------|
| Excel 기획서 | 104개 파일, 635시트 | **623시트 변환 (98.1%)** | Vision-First 파이프라인 |
| Confluence 문서 | 490페이지 | **489페이지 변환** | REST API 직접 추출 |
| PPT | 11개 | 1개 완료 | 후순위 |

### 핵심 기술: Vision-First 파이프라인

```
Excel/PDF 원본
    │
    ├── [1차] 스크린샷 캡처 (Headless)
    │         → 시트별 PNG 이미지 생성
    │
    ├── [2차] Claude Vision API 분석 (주 변환 엔진)
    │         → 레이아웃, 도형, 플로우차트, 이미지 내 텍스트 인식
    │         → 구조화된 Markdown 생성
    │
    ├── [3차] Excel 데이터 파싱으로 보강
    │         → 정확한 수치, 숨겨진 셀, 수식, 데이터 테이블
    │
    └── [합성] 두 결과 병합 → 최종 구조화된 Markdown
```

**성과**: 시범 변환에서 원본 2KB → 변환 결과 200KB (**100배 정보량 확보**)

## 4. 지금까지 만든 것 — 2단계: QnA Agent

### "기획에 대해 물어보면 정확하게 답한다"

**Agent 파이프라인 (5단계 처리)**:

```
질문 입력
    │
    ▼
[1] Planning (Sonnet)     질문 분석 → 검색 전략 수립
    │                      "어떤 워크북을 봐야 하는가?"
    ▼
[2] Search (4레이어)       구조적 + 지식그래프 + 벡터 + 풀텍스트
    │                      176개 워크북, 4,133 청크에서 최적 검색
    ▼
[3] Answer (Sonnet)       증거 기반 답변 생성
    │                      출처 명시, 표/다이어그램 포함
    ▼
[4] Reflection (Haiku)    자체 품질 검증
    │                      부족하면 재검색 1회 수행
    ▼
[5] 최종 답변              confidence 수준 표시 (high/medium/low/none)
```

### 검증 결과

| 지표 | 수치 | 비고 |
|------|------|------|
| **검색 정확도** | **97.2%** (481/495) | 규칙 기반 10차 평가 |
| **답변 품질** | **95.0%** (66/69) | LLM-as-Judge 8축 채점 |
| 트랩 질문 (할루시네이션 방어) | **100%** (9/9) | "없는 정보를 지어내지 않는가" |

**개선 이력**: 47% (기준선) → 72% → 87% → 94.2% → **95.0%**

### 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| LLM | Claude Sonnet / Haiku | AWS Bedrock 경유 |
| 임베딩 | Amazon Titan Embeddings v2 | 1024차원 |
| 벡터DB | ChromaDB | 4,133 청크 인덱싱 |
| 지식그래프 | NetworkX | 405 시스템, 627 관계 |
| Backend | FastAPI (Python) | SSE 스트리밍 지원 |
| Frontend | React + Vite (TypeScript) | SPA, localStorage 기반 |

## 7. 다음 단계 — 4단계: 기획 리뷰

### AI가 스스로 문제를 발견하고 알려주는 시스템

**질문을 기다리는 게 아니라, 먼저 보강이 필요한 기획서를 찾아 알려준다.**

### 리포트 유형

1. **충돌 리포트**
   - "A 기획서와 B 기획서가 같은 시스템에 대해 다른 정의를 사용합니다"

2. **공백 리포트**
   - "X 시스템은 기획서에서 참조되지만 상세 기획이 존재하지 않습니다"

3. **갱신 필요 리포트**
   - "이 기획서는 6개월간 미수정이지만, 관련 시스템 3개가 최근 변경되었습니다"
   - 보강 우선순위 자동 산출: 영향도 × 신선도 역수 × 변경 빈도

4. **버전 드리프트 리포트**
   - "이 시스템의 Excel 기획서(v3.2)와 Confluence 문서(v2.1)가 다릅니다"

### 신규 기획서 리뷰

- 기획서 업로드 → 기존 시스템과 정합성 자동 검토
- 누락된 고려사항 제안
- 유사 시스템 사례 비교

## 8. 다음 단계 — 5단계: 실시간 어시스턴트

### 기획자가 문서를 작성하는 순간 AI가 옆에서 도와준다

**UX 진화 로드맵**:

```
Phase 1  "물어보면 답함"           채팅 사이드바 (Chrome Extension)
    │     현재 페이지 자동 인식 + 관련 정보 표시
    ▼
Phase 2  "알아서 경고함"           인라인 경고
    │     작성 중인 내용이 기존 기획과 충돌 시 즉시 표시
    │     예: "스킬 시스템에는 기본 35%로 명시되어 있음"
    ▼
Phase 3  "직접 고쳐서 보여줌"      diff 제안 + 확인
         AI가 기획 지식 기반으로 구체적 텍스트 변경 제안
         기획자는 diff를 보고 수락/수정/거절
```

## 9. 확장 비전 — 역할별 AI 에이전트 (5-A단계)

**같은 지식 베이스 위에 페르소나만 바꾸면 모든 직군 지원 가능**

| 대상 | AI가 하는 일 | 예시 |
|------|-------------|------|
| **프로그래머** | 기획서 → 구현 가이드 | "이 기획을 구현하려면 어떤 코드를 수정해야 해?" |
| **QA** | 기획서 → 테스트케이스 자동 생성 | "변신 시스템 TC 작성해줘" → 정상/예외/경계값 목록 |
| **사업/운영** | 기획서 → 비개발자용 해설 | "패치노트 정리해줘", CS 답변 초안 자동 생성 |
| **아트** | 기획서 → 리소스 요구사항 추출 | "이 기획서에서 필요한 아트 에셋 목록 뽑아줘" |

## 10. 장기 비전 — 업무 프로세스 통합 (7~8단계, 후보)

### 기획 지식을 넘어, 업무 프로세스 전체를 AI가 이해

```
[현재]  기획서 데이터                    "이 시스템이 뭐야?"
    │
    ▼
[7단계] + JIRA + Perforce + Slack       "이 기획, 구현 상태가 어때?"
    │                                    "이 수치가 왜 바뀌었어?"
    │                                    "이 시스템 전문가 누구야?"
    ▼
[8단계] + 전사 커뮤니케이션 분석          "조직 간 정보 전달 병목은?"
                                         "R&R 회색지대는 어디야?"
```

### 교차 데이터 시너지

| 연결 | 가능해지는 것 |
|------|-------------|
| 기획서 + JIRA | 기획 구현 상태 자동 대시보드, 버그 패턴 분석 |
| 기획서 + Perforce 로그 | 밸런스 변경 타임라인, 무단 변경 감지 |
| 기획서 + Slack | 암묵지 발굴, "누가 전문가" 자동 파악, 버스 팩터 경고 |
| **전체 교차** | **의사결정 고고학** — 어떤 결정이든 Slack 논의 → 승인 → 문서화 → JIRA → 코드 커밋까지 전체 맥락 자동 복원 |

## 13. 피칭 시나리오 (5단계 완료 후)

| 순서 | 시연 내용 | 임팩트 |
|------|----------|--------|
| 1 | **QnA 데모**: "스킬 시스템과 변신 시스템이 어떻게 연동되나?" → 67개 기획서를 종합한 즉답 | 전체 기획 파악 가능 |
| 2 | **리뷰 데모**: 신규 기획서 업로드 → 기존 시스템과의 충돌/누락 자동 감지 | 재작업 비용 절감 |
| 3 | **프로액티브 데모**: "이번 주 보강이 필요한 기획서 Top 5" → 자동 리포트 | 사전 문제 감지 |
| 4 | **실시간 데모**: Confluence에서 기획서 작성 중 관련 정보 실시간 제안 | 작업 효율 향상 |
| 5 | **역할별 데모**: 같은 기획서를 개발자용/QA용/사업팀용으로 각각 변환 | 직군 확장성 |

### 피칭 포인트

- 신규 입사자 온보딩 시간 단축 (수 주 → 수 일)
- 기획 충돌/누락의 사전 감지 → 개발 재작업 비용 절감
- 핵심 인력 퇴사 시 암묵지 유실 방지
- 향후 게임 데이터 + 서버 코드까지 연결 시 기획↔구현 3중 교차 검증 가능
