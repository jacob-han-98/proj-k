# 시간 던전 시스템 (요약)

> 출처: 시스템 디자인 / 던전 / 시간 던전 시스템
> 원본: packages/confluence-downloader/output/시스템 디자인/던전/시간 던전 시스템/content.md

## 한 줄 설명
필드 외에 한시적으로 이용 가능한 사냥형 컨텐츠로, 시간정량형(체류 시간 기반)과 클리어형(입장 횟수 기반) 던전을 정의하고 입장 조건, 채널 관리, 보상 시스템을 규정한다.

## 핵심 용어
- 시간정량형 던전 (TimeDungeon)
- 클리어형 던전
- DungeonPlayTime (체류 시간)
- FreeTime (무료 시간)
- RechargedTime (충전 시간)
- TicketDungeon (입장권 소모형)
- 채널 (Channel)
- ChannelType (Fixed / Dynamic / 퍼시스턴트)
- MaxCapacity (최대 인원)
- ThresholdRate (채널 한계 인원)
- 입장 비용 (EntranceCost)
- DungeonLimitTime (던전 제한 시간)
- 클리어 조건 (DungeonClearCondition)
- 보상 (DungeonReward)
- 자동 퇴장
- 사망 페널티
- 월드 (World)
- 인스턴스 (Instance)
- 퍼시스턴트 (Persistent)

## 숫자/상수/공식
- 충전 시간 최대값: 9999시간
- Dynamic 채널 ThresholdRate 예시: 70% (MaxCapacity의 70% 달성 시 신규 채널 생성)
- Fixed 채널 예시: MaxCapacity = 500, ChannelCount = 2
- Dynamic 채널 예시: MaxCapacity = 50명, ThresholdRate = 70%
- 대표 보상 표시 개수: 5개 (TimeDungeon), 10개 (TimeDungeonLevelInfo)
- DungeonPlayTime 소모 순서: FreeTime → RechargedTime

## 참조 시스템
- PK_월드 시스템.xlsx
- ContentSetting (리셋 시간 공통 활용)
- WorldClass 테이블
- InstanceTrigger (클리어형 던전)
- MonsterSpawn / Monster / MonsterReward 테이블
- QuestReward 테이블 (구조 참고)
- PlayerConditionEnum
- CurrencyEnum
- CombatZoneTypeEnum

## 주요 섹션
- 개요 (K-던전 종류)
- 던전 구조 (던전 타입, 입장 조건, 월드, 채널)
- 던전 인게임 (시작 준비, 트리거, 클리어/보상, 퇴장, 사망/부활)
- 테이블 (TimeDungeon, TimeDungeonLevelInfo, TimeDungeonPrerequisite, WorldClass, TicketDungeon 참고용)
