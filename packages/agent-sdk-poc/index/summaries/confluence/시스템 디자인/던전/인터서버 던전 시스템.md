# 인터서버 던전 시스템 (요약)

> 출처: 시스템 디자인 / 던전 / 인터서버 던전 시스템
> 원본: packages/confluence-downloader/output/시스템 디자인/던전/인터서버 던전 시스템/content.md

## 한 줄 설명
동일 서버군의 유저들이 한 공간에 모이는 별도의 인터서버 던전 시스템으로, 입장 인원 제한, 오픈 시간, 체류 시간 제한 등의 장치를 통해 동시 접속 인원을 관리한다.

## 핵심 용어
- 인터서버 (통합서버)
- 동일 서버군
- 시간 던전
- 필드 사냥형
- 인터서버 전장
- 특수 던전
- ServerTypeEnum
- ServerInfo 테이블
- TimeDungeon 테이블
- GroupId
- MaxUserCount
- DungeonTypeEnum
- MaxUserPerServer
- 크로스월드
- 지역 채팅
- 서버 침공
- 서버 탐험
- 네임플레이트
- 파티 탈퇴
- 길드 초대
- 망 단절
- UTC 시간

## 숫자/상수/공식
- 한 던전 입장 인원 제한 예시: 1000명 (오딘)
- 오픈 시간 예시: 22:00~00:00
- 체류 시간 제한 예시: 3시간
- 컨텐츠별 서버별 입장 인원 제한 예시: A 컨텐츠 100명, B 컨텐츠 200명
- 서버 번호 표시 예시: 소르브1~소르브10
- 다중 오픈 시간 예시: 매일 11:00~13:00, 22:00~00:00

## 참조 시스템
- 서버 그룹 & 매칭 규칙 정의
- 서버 침공 상세 기획

## 주요 섹션
- 개요
- 규칙 (플레이 양상, 서버 세팅, 던전 입장, 퇴장, 컨텐츠 제한, 파티, 길드, 인터서버 채팅)
- UI (네임플레이트)
- 테이블 (서버 리스트 제어, ServerInfo, TimeDungeon)
- 추후 개발 (던전 오픈 시간, 망 단절 시)
