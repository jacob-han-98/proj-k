# 컬렉션 시스템 (요약)

## 한 줄 설명
지정된 아이템을 세트에 등록하여 완성하면 스탯 보상을 얻는 시스템으로, 획득한 스탯 효과는 동일 계정 내 동일 서버 캐릭터들이 공유한다.

## 핵심 용어
- 컬렉션 세트
- 아이템 등록
- 스탯 보상
- 달성도 공유
- 타이틀
- 요구 아이템 정보
- 보상 스탯 정보
- 가방
- 착용 중 아이템
- 잠긴 아이템
- 아이템 그룹 (그룹 ID)
- 카테고리 분류
- 대분류 / 소분류
- CollectionCategoryGroupEnum
- CollectionCategoryEnum
- 즐겨찾기 기능
- 숨기기 기능
- 레드닷
- 강화 단계
- 거래소
- 일괄 등록 기능
- 아이템 검색 기능
- 필터 기능

## 숫자/상수/공식
- 컬렉션 세트 구성: 1~6개 아이템
- 컬렉션 세트 ID: 6자리 (예: 100000)
- 보상 스탯: 복수개의 1차 및 2차 스탯효과 세팅 가능 (EffectStatName01~05)
- 장비 수량: 1개만 (TargetItem이 ItemEquipClass인 경우)
- 비장비 수량: 1개 이상 기입 가능
- 리스트 정렬 기준 순서: 미완료→완료, 일반→숨김, CategoryEnum순, 달성가능→강화미달→모두미달, 등급낮음→높음, 티어낮음→높음, 강화단계낮음→높음, ID순

## 참조 시스템
- 아이템 시스템 (ItemClass, ItemEquipClass, ItemEquipType, ItemConsumeType, ItemEtcType)
- 강화 시스템
- 거래소 시스템
- 스탯 시스템 (StatClass, BonusEnum)
- 이벤트 스케줄러 테이블

## 주요 섹션
- 개요
- 플레이 흐름
- 규칙 (기본 구조, 등록 조건, 리스트, 보상 획득, 아이템 등록 유도 안내)
- 연계된 시스템 (아이템 툴팁, 강화 시스템, 거래소)
- 부가 편의 기능 (일괄 등록, 아이템 검색, 필터, 자동 강화)
- 테이블
- UI
