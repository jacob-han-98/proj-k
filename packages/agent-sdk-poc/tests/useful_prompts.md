# Project K — 유용성 체크리스트 (기획자 실사용 질문)

> 이 파일은 agent-sdk-poc 품질을 **정답 맞추기가 아니라 유용성**으로 평가하기 위한 질문 목록이다.
> 각 질문의 체크리스트는 "답변이 담고 있으면 기획자에게 도움이 되는 정보"다.
> 모든 항목을 만족하지 않아도 되지만, 핵심 항목(★) 은 반드시 들어가야 한다.
> 평가 방식: `tests/compare.py`로 qna-poc vs agent-sdk-poc 답변을 나란히 생성 → `tests/runbook.md`에 수기 체크.

## 평가 지침

- **정량 점수 금지**: "X/Y 정답" 식으로 매기지 않는다. 체크리스트는 **트래킹용**.
- **비교 컨텍스트**: 같은 질문을 qna-poc와 agent-sdk-poc에 보내 답변을 나란히 저장 → 기획자(사용자) 또는 LLM-as-judge가 "어느 쪽이 실제 업무에 더 유용한가"만 본다.
- **질문은 살아 있는 문서**: 운영하면서 새 질문 유형이 발견되면 이 파일에 추가한다. 지엽적 정답보다 **다양한 사용 패턴 커버**가 우선.
