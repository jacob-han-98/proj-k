## 3. 검증 방법론

### 3.1 단계별 검증 (프로젝트 방침 준수)

```
[1] --dry-run: 연결 확인 + 페이지 트리 구조 파악
    → 147개 페이지, 5단계 깊이 확인 ✓
    ↓
[2] 1개 페이지 시범 다운로드 (텍스트 전용)
    → 변환 품질 확인: 프론트매터, 리스트, 구조 ✓
    ↓
[3] 1개 페이지 시범 다운로드 (이미지 포함)
    → 이미지 다운로드 + 참조 정확성 확인 ✓
    ↓
[4] 전체 다운로드 실행
    → (대기 중)
    ↓
[5] 변환 품질 검증: 테이블, 코드블록, 매크로 등 다양한 요소
    → (대기 중)
```

### 3.2 검증 기준

| 항목 | 기준 | 검증 방법 |
|------|------|-----------|
| 다운로드 성공률 | ≥ 95% | download_results.json 집계 |
| 이미지 참조 유효성 | 100% | MD 내 `![](images/...)` ↔ 실제 파일 매칭 |
| 프론트매터 완전성 | 5개 필드 전부 | 모든 content.md 검사 |
| 폴더 구조 정확성 | Confluence 트리와 1:1 | _manifest.json vs 실제 폴더 대조 |
| 텍스트 보존률 | 원본 Storage Format 텍스트 100% | 원본 페이지와 대조 |
| 매크로 변환율 | 주요 매크로 100% | code, info, panel, expand, image |

## 4. Stage별 검증 현황

### 4.1 Stage 1: Crawl

| 검증 항목 | 기준 | 현황 |
|-----------|------|------|
| API 연결 | 인증 성공 | ✓ |
| 페이지 트리 구축 | 재귀 탐색 완료 | ✓ (147개) |
| 페이지네이션 | 100개 초과 하위 페이지 | 구현됨 (미발생) |
| 503 재시도 | 서버 오류 복구 | ✓ (1회 발생, 자동 복구) |
| 실시간 진행 | 탐색 중 표시 | ✓ |

### 4.2 Stage 2: Download

| 검증 항목 | 기준 | 현황 |
|-----------|------|------|
| 페이지 본문 | storage format 추출 | ✓ |

| 에러 핸들링 | 개별 실패 → 전체 계속 | 구현됨 |

### 4.3 Stage 3: Convert

| 검증 항목 | 기준 | 현황 |
|-----------|------|------|
| 표준 HTML 변환 | p, h1-6, strong, em, ul, ol, table, a | ✓ (markdownify) |
| ac:image | 이미지 참조 변환 | ✓ |
| code 매크로 | 코드 블록 + 언어 | 구현됨 (미검증) |
| info/note/warning/tip | blockquote 변환 | 구현됨 (미검증) |
| panel 매크로 | blockquote + 제목 | 구현됨 (미검증) |
| expand 매크로 | details/summary | 구현됨 (미검증) |
| ac:link | 내부 링크 변환 | 구현됨 (미검증) |
| task-list | 체크박스 변환 | 구현됨 (미검증) |
| toc 매크로 | 제거 | 구현됨 |
| 동적 매크로 | 제거 | 구현됨 |

> "미검증": 해당 요소를 포함한 페이지를 아직 다운로드하지 않음. 전체 실행 후 확인 필요.

## 6. 알려진 한계

| 한계 | 설명 | 심각도 |
|------|------|--------|
| 동적 매크로 손실 | children, jira, include 등 런타임 데이터 제거 | 낮음 |
| 서드파티 매크로 | draw.io, Gliffy 등 미지원 | 중간 (해당 시 추가 구현) |
| 내부 링크 정확도 | 페이지 간 상호 링크가 `#title` 형태로만 변환 | 낮음 |
| 비이미지 첨부 | PDF, 동영상 등은 다운로드 안 함 | 낮음 |
| 테이블 복잡도 | 병합 셀 등 복잡한 테이블은 markdownify 한계 | 중간 |
| 권한 제한 페이지 | API 토큰 권한 밖 페이지 접근 불가 | 낮음 |
