## 3. 검증 방법론

### 3.1 단계별 검증 (프로젝트 방침 준수)

```
[1] --dry-run: 연결 확인 + 페이지 트리 구조 파악
    → 490개 페이지 (page+folder), 7단계 깊이 확인 ✓
    ↓
[2] 1개 페이지 시범 다운로드 (텍스트 전용)
    → 변환 품질 확인: 프론트매터, 리스트, 구조 ✓
    ↓
[3] 1개 페이지 시범 다운로드 (이미지 포함)
    → 이미지 다운로드 + 참조 정확성 확인 ✓
    ↓
[4] 전체 다운로드 실행
    → 490/490 성공 (100%) ✓
    ↓
[5] 변환 품질 검증: 테이블, 코드블록, 매크로 등 다양한 요소
    → (다음 작업)
```

### 3.2 검증 기준

| 항목 | 기준 | 검증 방법 |
|------|------|-----------|
| 다운로드 성공률 | ≥ 95% | download_results.json 집계 |
| 이미지 참조 유효성 | 100% | MD 내 `![](images/...)` ↔ 실제 파일 매칭 |
| 프론트매터 완전성 | 5개 필드 전부 | 모든 content.md 검사 |
| 폴더 구조 정확성 | Confluence 트리와 1:1 | _manifest.json vs 실제 폴더 대조 |
| 텍스트 보존률 | 원본 Storage Format 텍스트 100% | 원본 페이지와 대조 |
| 매크로 변환율 | 주요 매크로 100% | code, info, panel, expand, image |

## 6. 알려진 한계

|------|------|--------|
| 동적 매크로 손실 | children, jira, include 등 런타임 데이터 제거 | 낮음 |
