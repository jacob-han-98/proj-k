**» 각 아이콘 클릭 시 세부 UI 출력**
→ 시간은 "[0]일 [0]시간 [0]분 [0]초"까지 출력
→ 단, 값이 0인 단위는 미출력
→ "액티브"탭이 기본 선택된 상태로 출력
→ 한 번에 6.5개 출력 + 해당 영역은 위아래 스크롤 가능
→ **BuffClass - Duration값이 -1이면 "패시브"탭에 출력**

**» 세부 UI 내의 돋보기 아이콘 터치 시 효과 UI 출력**
→ 효과 UI 크기는 고정
→ 효과 개수에 따라 크기 변화 X
→ 한 번에 최대 4.5개까지 출력 + 해당 영역은 위아래 스크롤 가능

![버프/디버프 세부 UI 스크린샷](./images/HUD_기본_detail_r1_fig1.png)
- 좌측: 버프 목록 UI (액티브/패시브 탭, 버프 아이콘 + 버프 명칭 + 잔여시간 리스트)
- 우측: 선택한 버프의 효과 상세 UI
  - **BuffClass - TextkeyTitle 정보 출력**
  - **BuffClass - TextkeyDesc 정보 출력**
  - **EffecClass에서 StatClass에 등록된 정보일 경우 해당 TextKey와 입력한 수치 출력** (Ex. AllAttack, MaxHp 등등)
  - **EffecClass에서 StatClass에 등록되지 않은 정보일 경우 공란으로 출력** 이때는 TextkeyDesc를 연결하여 설명 필요

---

**» 버프 아이콘 출력 예외 사항**
→ 하나의 버프에 2개 이상의 Effect가 연결되어 있을 때 그 중 하나라도 대상에게 적중을 하지 않으면 나머지 효과도 미적중 처리
  - ✓ Ex. A버프(효과a, 효과b 연결)를 B보스 몬스터에 걸었으나 B보스가 효과a에 면역이면 효과b는 면역 여부와 상관 없이 적용하지 않음
  - ✓ 위와 같은 경우는 기획적으로 발생할 일이 매우 드물지만 편의상 위와 같이 하기로 병호님과 논의 함 (24.11.04)
  - ✓ 추후 각 효과를 개별적으로 걸리도록 할 때는 병호님께 요청 필요

• (2024.12.08) 중첩(스택)이 되는 버프 관련 표현 규칙은 아래 경로 참조
→ PK_타게팅 시스템.xlsx - "UI" 시트 - "[버프 중첩 표현]"

### ⑪ 신규 유저 카테고리명 정의

**» 신규 유저가 컨텐츠명만 봐도 어떤 컨텐츠인지 예상이 가능하도록 UI에 다음과 같은 카테고리명을 통해 정립**

- **성장**: 캐릭터 성장에 직접적으로 영향을 주는 컨텐츠를 모음 (Ex. 변신, 도감)
- **컨텐츠**: 유저가 직접 이용 및 참여하는 컨텐츠를 모음 (Ex. 인던, 채집)
- **커뮤니티**: 유저 간 직간접적인 소통의 매개가 되는 컨텐츠를 모음 (Ex. 길드, 랭킹)
- **편의&기타**: 위의 3종 이외의 모든 기능을 모음

**» 각 카테고리 구분 경계 및 전체 UI 공간은 해당 아이콘 수량에 따라 자동 조절**
- "컨텐츠" 카테고리 아이콘이 1개 미만일 시 "컨텐츠" 카테고리 UI 및 해당 공간은 출력되지 않음 (아래 우측 예시 이미지 참조)

→ 커뮤니티 : 유저 간 직/간접적인 소통의 매개가 되는 컨텐츠들 모음 (Ex. 길드, 랭킹)
→ 편의&기타 : 위의 3종 이외의 모든 기능들 모음
**» 각 카테고리 구분 경계 및 전체 UI 공간은 해당 아이콘 수량에 따라 자동 조절**
→ "컨텐츠" 카테고리 아이콘이 1개 미만일 시 "컨텐츠" 카테고리 내 명 제목 고가도 출력되지 않음 (아래 오른 예시 이미지)
