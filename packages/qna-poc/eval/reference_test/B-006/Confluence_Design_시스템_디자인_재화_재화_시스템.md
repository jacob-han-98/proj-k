> # 1. 개요

- 아이템 구매/판매/제작 및 각종 컨텐츠 및 기능 이용 시 교환되는 재화 단위를 의미
- 크게 "유료 재화"와 "무료 재화"로 구분

  - 유료 재화 = BM 등 게임 외부에서 획득한 재화

    - 유저 간의 거래로 인해 획득한 재화도 출처는 BM이므로 유료재화로 간주
  - 무료 재화 = 인게임에서 생산되는 재화

> # 2. 세부 정보

### [1] 재화의 귀속 여부

- 재화는 캐릭터 / 서버 / 길드 기준으로 귀속됨

  - 캐릭터 귀속 - 해당 캐릭터만 이용 가능, 캐릭터 삭제 시 같이 삭제
  - ****서버 귀속**** - 해당 서버에서만 이용 가능, 타 서버에 캐릭터 생성 시 재화 보유량은 타 서버의 기준을 따름

    - 해당 서버의 캐릭터를 모두 삭제했다가 재생성 시에도 남아 있는 재화
    - <계정 귀속>과 혼용되면 안됨 - 계정 내에 여러 서버의 캐릭터 존재 가능
  - 길드 귀속 - 해당 길드에서만 이용 가능, 길드 삭제 시 같이 삭제

### [2] 유료 재화 타입 분리

- 유료 재화는 기본적으로 현금으로 구매해야 하지만, 유저 간 거래를 통해서도 획득 가능하기 때문에 재화 타입 분리 필요

  - 유상/무상 재화 분리해야 환불 등의 법적 이슈 대응 가능
  - 유저가 보기에는 다 같은 재화이지만, 내부적으로 타입이 분리되어 있는 상태

    - 따라서, UI 에서 이 부분을 구분하여 보여줄 필요가 없으며, 하나의 재화로 UI에서 표현
  - 유료 재화 구분

    - 유저에게 지급되는 타입 - BM 구매 획득 (유료) / 컨텐츠용 (무료)

      - Enum으로 분리

> # 3. UI

- HUD 상단 기조

  - 평소에는 기본적인 재화 (유료재화, 골드)만 노출
  - 컨텐츠 UI에서는 컨텐츠 이용에 필요한 재화 노출
  - BM 없는 재화 아이콘 사용

    - Currency 테이블에 아이콘 경로 추가
- 재화의 정보 툴팁 표현

  - 재화를 보상으로 받았을 때나 정보 열람을 하기 위한 툴팁
  - ItemEtcClass에 아이템 툴팁 형태를 빌려 쓸 수 있는 가데이터 추가

    - ItemType = Dummy
  - CurrencyClass에 툴팁 띄울 Item ID 연결 컬럼 추가

### RewardTypeEnum

- 재화가 늘어남에 따라 Currency가 추가되면 RewardType도 같이 추가
- Reward 테이블 등에서의 유료 재화 기입 규칙:

  - 타입에 따라 RewardTypeEnum = Diamond / FreeDiamond 지급
- Merchant 테이블 등 유저가 소모해야 하는 케이스에서는 Diamond/FreeDiamond 중 아무거나 기입

  - 서버에서 소모 순서대로 차감할테지만 기획에서는 가급적 Diamond로 통일하여 기재
