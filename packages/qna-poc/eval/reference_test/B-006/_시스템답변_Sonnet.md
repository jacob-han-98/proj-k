# 컨텐츠 재화 상인 상점 UI/UX 설계 포인트

## 1. 상점 UI 기본 구조 (상인 NPC 시스템 기반)

### 1-1. 공통 UI 레이아웃
```
┌─────────────────────────────────────────┐
│ [상점 구분명 + Icon]              [X]   │ ← NpcClass_IconField 리소스 사용
├─────────────────────────────────────────┤
│ ② 상품 진열부        │ ③ 인벤토리부      │
│ (좌측)               │ (우측)            │
│                      │                   │
│ [아이템 그리드]      │ [인벤토리 그리드] │
│                      │                   │
├─────────────────────────────────────────┤
│ 보유 재화: 💎999,999 🪙9,999,999,999    │
│           [컨텐츠 재화] 999,999         │ ← 해당 상인의 거래 재화만 표시
└─────────────────────────────────────────┘
```

**핵심 설계 원칙:**
- 상점 UI가 켜진 상태에서는 **캐릭터 조작 금지** (안전 지대 전제)
- 상인 NPC 소멸 시 → UI 클로징 + 상점 인터랙션 종료
- 인벤토리부는 **구매 완료 시 갱신만 사용** (판매 기능은 일반 상인 전용)

## 2. 컨텐츠 재화별 상점 특화 설계

### 2-1. 4종 상인별 거래 재화 및 판매 품목

| 상인 | 거래 재화 | 판매 품목 카테고리 | NpcSubCategory |
|------|-----------|-------------------|----------------|
| **길드 상인** | 길드 주화 | PVE 소모성 (문학주문서, 소환서, 스킬북, 성장 재료, 제작 재료) | Guild |
| **일일미션 상인** | 성장의 증표 | PVE 소모성 (동일) | Achievement |
| **무한의 탑 상인** | 정복의 증표 | PVE 희귀 재료, 스킬북, 성장 재료, 제작 재료 | InfinityTower |
| **레이드 상인** | 승리의 훈장 | PVP 관련 (강화제 보구류, 고급 재료, 특수 장비) | Raid |

**재화 표시 우선순위:**
1. 해당 상인의 **거래 재화**를 가장 크게 표시
2. 골드/다이아는 보조 정보로 우측 상단에 축소 표시
3. 다른 컨텐츠 재화는 **표시하지 않음** (혼란 방지)

### 2-2. 재화 부족 시 UX 처리

**Case 1: 컨텐츠 재화 부족**
```
[시스템 메시지]
"[재화명]이(가) 부족합니다."
(예: "길드 주화가 부족합니다.")

→ 별도 구매 페이지 이동 팝업 없음
→ 해당 컨텐츠 안내 메시지 추가 권장
   "길드 활동을 통해 획득할 수 있습니다."
```

**Case 2: 골드 부족 (일부 상인이 골드도 함께 요구하는 경우)**
```
[구매 페이지 이동 팝업]
"재화가 부족합니다. 구매 화면으로 이동하시겠습니까?"
부족 금액: 💎 3,000
[취소] [확인]
```

**설계 포인트:**
- 컨텐츠 재화는 **게임 플레이로만 획득 가능** → BM 상점 연결 불필요
- 골드/다이아 부족 시에만 구매 페이지 이동 제공

## 3. 인벤토리 탭 구조와 자동 분류 로직

### 3-1. 인벤토리 탭 구조 (기획서 기준)

```
[인벤토리 UI]
┌─────────────────────────────────────┐
│ 가방                          [X]   │
├─────────────────────────────────────┤
│ [All] [⚔️장비] [💊소모품] [🌿재료] [A자동] │ ← 탭 메뉴
├─────────────────────────────────────┤
│                                     │
│     [아이템 그리드 슬롯]            │
│                                     │
├─────────────────────────────────────┤
│ 50 / 100                      [+]   │
│ [정렬] [휴지통]                     │
└─────────────────────────────────────┘
```

**탭별 필터 규칙:**
- **All**: 모든 ItemTypeEnum 표시
- **⚔️장비**: `ItemTypeEnum = Equip`
- **💊소모품**: `ItemTypeEnum = Consume`
- **🌿재료**: `ItemTypeEnum = Material`
- **A자동**: 자동 사용 중인 아이템 (`자동 사용 중 UI` 활성화된 아이템)

---

### 3-2. 구매 후 자동 분류 로직

#### **Step 1: 구매 완료 시 처리**
```
1. 선택 아이템의 총 비용만큼 재화 감소
2. 선택 아이템의 총 수량만큼 인벤토리로 이동
   → ItemTypeEnum에 따라 자동 분류
3. 구매 성공 메시지 출력
   "(아이템명)을 ??개 구매하였습니다."
```

#### **Step 2: 인벤토리 슬롯 배치 규칙**
```mermaid
flowchart TD
    A[아이템 구매 완료] --> B{ItemTypeEnum 확인}
    B -->|Equip| C[장비 탭 영역에 배치]
    B -->|Consume| D{ConsumeTypeEnum 확인}
    B -->|Material| E[재료 탭 영역에 배치]
    
    D -->|Potion/Cook| F[소모품 탭 영역에 배치]
    D -->|EnchantWeapon/Armor/Accessory| G[재료 탭 영역에 배치<br/>강화 재료로 분류]
    D -->|ItemRandomBox/ChoiceBox| F
    
    C --> H{CanStack 확인}
    F --> H
    E --> H
    G --> H
    
    H -->|CanStack=True & 동일 아
