# xlsx-extractor 검증 프로토콜

> 각 Stage별 검증 방법과 최종 품질 기준을 정의한다.

---

## 1. Stage별 단위 검증

### Stage 1: Capture 검증

| 검증 항목 | 방법 | PASS 기준 |
|-----------|------|-----------|
| PDF 생성 | 시트 수 == PDF 수 | 100% 일치 |
| PNG 변환 | PDF 페이지 수 == PNG 수 (빈 페이지 제외) | 100% 일치 |
| 빈 페이지 감지 | 98% 백색 임계값 적용 | 실제 빈 시트만 건너뛰기 |
| 이미지 크기 | overview.png <= 1568px 너비 | 100% 준수 |
| 분할 이미지 크기 | detail_*.png <= 1568x1568px | 100% 준수 |
| 오버랩 | 인접 타일 경계 10-15% 겹침 | 육안 확인 (경계선의 텍스트 잘림 없음) |
| 콘텐츠 크롭 | 불필요한 여백 제거 | 콘텐츠 손실 없이 여백 최소화 |

**검증 스크립트 (향후 구현)**:
```
python -m xlsx_extractor.verify stage1 --input <xlsx> --output-dir <dir>
```

### Stage 2: Vision 검증

| 검증 항목 | 방법 | PASS 기준 |
|-----------|------|-----------|
| API 호출 성공 | HTTP 200 응답 | 100% (재시도 포함) |
| 2-이미지 전달 | 요청 body에 image 2개 포함 확인 | 모든 호출 |
| 텍스트 생성 | 빈 응답 아닌지 확인 | 비어있지 않음 |
| 테이블 구조 | Markdown 테이블 파싱 가능 여부 | 헤더+구분선+행 구조 |
| Mermaid 구조 | Mermaid 코드 블록 유효성 | mermaid-cli 또는 수동 확인 |
| 서브 이미지 마킹 | `[IMAGE: ...]` 마커 존재 여부 | 해당되는 시각 요소에 마킹 |
| 연속성 | 이전 타일과 중복/누락 없는지 | 타일 간 테이블 행 번호 연속 |

### Stage 3: Parse 검증

| 검증 항목 | 방법 | PASS 기준 |
|-----------|------|-----------|
| 셀 데이터 | openpyxl 읽기 성공 | 에러 없음 |
| 숨겨진 행/열 | hidden=True 행/열 감지 | 실제 숨겨진 항목과 일치 |
| 수식 추출 | data_only=False로 수식 원본 | 수식 셀 수 == 추출 수 |
| 병합 셀 | merged_cells.ranges 확인 | 병합 영역 누락 없음 |

### Stage 4: Synthesize 검증

| 검증 항목 | 방법 | PASS 기준 |
|-----------|------|-----------|
| 최종 MD 생성 | content.md 파일 존재 | 비어있지 않음 |
| 서브 이미지 저장 | images/ 폴더 내 파일 수 == [IMAGE] 마커 수 | 100% 일치 |
| 이미지 참조 | MD 내 `![...]` 링크의 파일이 실제 존재 | 모든 링크 유효 |
| 오버랩 중복 제거 | 동일 테이블 행 중복 없음 | 행 번호/내용 기준 검사 |
| openpyxl 보강 적용 | 숨겨진 데이터 섹션 존재 | 숨겨진 행/열이 있는 경우 |

## 2. Vision AI 랜덤 질의 검증 (핵심)

변환 결과의 정확성을 Vision AI 자체로 검증하는 시스템.

### 2.1 검증 프로세스

```
content.md에서 검증 가능한 항목 추출
  |
  v
랜덤으로 3-5개 항목 선택
  |
  v
각 항목에 대해 질문 생성
  |
  v
원본 이미지 + 질문 -> Vision AI에 전달
  |
  v
Vision AI 응답 vs 추출 결과 비교
  |
  v
정답률 계산 -> PASS/FAIL 판정
```

### 2.2 질문 유형

| 유형 | 예시 질문 | 검증 대상 |
|------|-----------|-----------|
| 테이블 값 | "이 이미지에서 3번째 행의 '단축키' 열 값은?" | 셀 값 정확도 |
| 테이블 구조 | "이 테이블의 열 개수와 열 이름은?" | 테이블 구조 |
| 플로우차트 노드 | "이 흐름도에서 '시작' 다음 단계는?" | 흐름도 해석 |
| 텍스트 존재 여부 | "이 이미지에 '{특정 텍스트}'가 있습니까?" | 텍스트 추출 누락 |
| 숫자 정확도 | "HP 바에 표시된 수치는?" | 수치 정확도 |
| 위치/레이아웃 | "표의 위에 있는 제목 텍스트는?" | 레이아웃 해석 |

### 2.3 질문 생성 규칙

1. content.md를 파싱하여 검증 가능 항목 목록 생성:
   - 테이블의 각 셀 값
   - 플로우차트 노드/엣지
   - 제목/소제목 텍스트
   - 숫자/수치 데이터

2. 항목 중 **랜덤 3-5개** 선택 (시트 크기에 비례):
   - 소형 시트 (타일 1개): 3개
   - 중형 시트 (타일 2-6개): 4개
   - 대형 시트 (타일 7+개): 5개

3. 각 항목에 대해 구체적 질문 생성:
   ```
   "이 이미지를 보고 답하세요:
    {질문}
    한 줄로 정확히 답하세요."
   ```

### 2.4 검증 판정

| 정답률 | 판정 | 후속 조치 |
|--------|------|-----------|
| 80%+ | **PASS** | 변환 완료 |
| 60-79% | **WARN** | 경고 플래그, 수동 리뷰 권장 |
| 60% 미만 | **FAIL** | 해당 시트 재변환, 그래도 FAIL 시 수동 검토 |

### 2.5 검증 결과 저장

```json
{
  "sheet_name": "HUD",
  "verification_date": "2026-03-07T12:00:00",
  "model": "claude-opus",
  "questions": [
    {
      "question": "이 테이블에서 번호 13의 기능은?",
      "expected": "자동 물약 슬롯",
      "actual": "자동 물약 슬롯",
      "match": true,
      "source_tile": "detail_r1_c0"
    },
    {
      "question": "화면 하단 중앙의 큰 원형 버튼의 번호는?",
      "expected": "19",
      "actual": "19",
      "match": true,
      "source_tile": "detail_r1_c1"
    }
  ],
  "score": 4,
  "total": 5,
  "accuracy": 0.80,
  "verdict": "PASS"
}
```

## 3. 통합 검증 (End-to-End)

### 3.1 프로토타입 테스트 대상

| 파일 | 시트 수 | 특성 | 검증 목적 |
|------|---------|------|-----------|
| PK_단축키_시스템.xlsx | 3 | 테이블 중심, UI 스크린샷 포함 | 기본 파이프라인 |
| PK_변신_및_스킬_시스템.xlsx | 13 | 대형, 플로우차트 다수 | 대용량/복잡 도형 |
| PK_버프_시스템.xlsx | (확인 필요) | 수치 테이블 중심 | 수치 정확도 |

### 3.2 레거시 결과 비교

기존 변환 결과(`_knowledge_base/sheets/`)와 새 결과를 비교:

| 비교 항목 | 방법 |
|-----------|------|
| 정보량 | 문자 수, 테이블 행 수 비교 |
| 테이블 정확도 | 동일 테이블의 셀 값 일치율 |
| 도형/플로우차트 | 노드/엣지 수 비교 |
| 이미지 활용 | 서브 이미지 수 및 텍스트 참조 유무 |

### 3.3 자동화 검증 파이프라인

```
python -m xlsx_extractor.verify full \
    --input <xlsx> \
    --output-dir <dir> \
    --compare-legacy <legacy_md>  # 선택: 레거시 결과 비교
```

출력:
- `_meta/verification.json` (질의 검증 결과)
- `_meta/extraction_log.json` (각 Stage 소요시간, 에러)
- 콘솔에 PASS/WARN/FAIL 요약

| 항목 | 기준 |
|------|------|
| 시트 캡처 성공률 | >= 95% (3시트 중 1 실패까지 허용) |
| Vision AI 해석 성공률 | 100% (실패 시 재시도) |
| 랜덤 질의 정답률 | >= 80% (PASS) |
| 서브 이미지 참조 유효성 | 100% (깨진 링크 없음) |
| 오버랩 중복 제거 | 동일 행 중복 0건 |
| openpyxl 보강 적용 | 숨겨진 데이터 존재 시 100% |
